## Accessibility

The `Progress` component follows accessibility best practices:

- Uses semantic progressbar role with proper ARIA attributes
- Provides `aria-valuemin`, `aria-valuemax`, and `aria-valuenow` for screen readers
- Maintains visual clarity with proper contrast ratios
- Supports customization without breaking accessibility features

## Best Practices

- Use progress bars for tasks with a known completion percentage
- Provide contextual text labels to describe what the progress represents
- Consider using appropriate heights based on the importance and visibility needs
- The component works seamlessly in both light and dark modes
- For indeterminate progress (unknown duration), consider using a loading spinner instead
