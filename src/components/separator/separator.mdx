## Accessibility

The `Separator` component follows accessibility guidelines:

- Decorative separators are hidden from screen readers by default
- Non-decorative separators are announced to assistive technologies
- Proper ARIA attributes for semantic meaning
- No keyboard interaction required (visual element only)

## Best Practices

- Use separators to create visual hierarchy and group related content
- Choose appropriate orientation (horizontal vs vertical) for your layout
- Consider spacing around separators for optimal visual balance
- Use semantic separators when the division has meaning beyond visual design
- Keep separator styling consistent throughout your application
- Test with screen readers to ensure appropriate announcements
