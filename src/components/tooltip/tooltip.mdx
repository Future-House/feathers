## Accessibility

The `Tooltip` component follows the [WAI-ARIA Tooltip Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tooltip/) and includes:

- Keyboard focus support with Escape key to close
- Screen reader compatibility with proper ARIA attributes
- Focus management that doesn't trap keyboard users
- Support for both hover and focus interactions
- Automatic collision detection and repositioning

## Best Practices

- Keep tooltip content concise and helpful
- Use tooltips for supplementary information, not essential content
- Ensure tooltips don't interfere with user interactions
- Test with keyboard navigation and screen readers
- Consider mobile users who cannot hover
- Use appropriate delays to avoid overwhelming users
- Position tooltips to avoid covering important content
- Provide alternative ways to access tooltip information on touch devices
