## Accessibility

The `Slider` component follows the [WAI-ARIA Slider Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/slider/) and includes:

- Keyboard navigation support with arrow keys, Home, and End keys
- Screen reader announcements for current value and range
- Proper ARIA attributes including `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`
- Focus management and visual focus indicators
- Support for custom ARIA labels and descriptions

## Best Practices

- Always provide accessible labels for slider components
- Use appropriate step values that match your use case
- Consider the range of values and choose reasonable min/max boundaries
- Test with keyboard navigation to ensure accessibility
- Use `onValueCommit` for expensive operations that shouldn't run on every change
- Provide visual feedback for disabled states
