## Features

- **Multiple Modes**: Support for date-only, time-only, or combined date-time selection
- **Flexible Time Format**: 12-hour (with AM/PM) or 24-hour format
- **Seconds Support**: Optional seconds selection
- **Minute Steps**: Configurable minute intervals (e.g., 5, 10, 15 minutes)
- **Fully Accessible**: Built on top of accessible Radix UI primitives
- **Customizable**: Extensive props for formatting and styling
- **Dark Mode**: Full dark mode support

## Usage

```tsx
import { DateTimePicker } from '@future-house/feathers';

function Example() {
  const [date, setDate] = useState<Date | undefined>();

  return (
    <DateTimePicker
      value={date}
      onChange={setDate}
      placeholder="Select date and time"
    />
  );
}
```

## Accessibility

The DateTimePicker component is built with accessibility in mind:

- Full keyboard navigation support
- ARIA labels and roles
- Screen reader friendly
- Focus management
- Escape key to close popover

## Best Practices

1. **Provide Clear Labels**: Always provide a clear label or placeholder to indicate what the user should select
2. **Use Appropriate Mode**: Choose the right mode (date, time, or datetime) based on your use case
3. **Consider Time Zones**: Be aware of time zone considerations when working with dates
4. **Validation**: Implement proper validation for date ranges or specific date requirements
5. **Format Consistently**: Use consistent date/time formats throughout your application
