## Accessibility

The `Pagination` component follows accessibility best practices:

- Uses semantic `nav` element with proper ARIA labels for navigation context
- Provides proper `aria-current="page"` attribute for the active page
- Supports keyboard navigation with Tab, Enter, and Space keys
- Uses descriptive `aria-label` attributes for Previous/Next buttons
- Provides screen reader accessible text for ellipsis elements
- Maintains proper focus management throughout navigation
- Uses appropriate button and link semantics for different interactions

## Best Practices

- Use clear and descriptive labels for navigation buttons
- Implement proper page state management in your application
- Consider the total number of pages when choosing pagination layout
- Provide visual feedback for the current page state
- Use ellipsis appropriately to avoid overwhelming users with too many page links
- Ensure adequate spacing between clickable elements for touch interfaces
- Test with keyboard navigation and screen readers
- Consider mobile responsiveness for smaller screens
- Implement proper loading states during page transitions
- Use consistent pagination patterns throughout your application
