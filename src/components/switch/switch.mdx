## Accessibility

The `Switch` component follows the [WAI-ARIA Switch Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/switch/) and includes:

- Keyboard navigation support with Space and Enter keys
- Screen reader announcements for state changes
- Proper ARIA attributes including `aria-checked` and `aria-labelledby`
- Focus management and visual focus indicators
- Support for `aria-describedby` for additional context

## Best Practices

- Always provide clear labels for switch components
- Use switches for binary on/off states, not for selection between options
- Group related switches logically in forms and settings
- Provide immediate visual feedback for state changes
- Test with keyboard navigation and screen readers
- Use descriptive text to explain what the switch controls
- Consider the impact of changes and whether confirmation is needed
