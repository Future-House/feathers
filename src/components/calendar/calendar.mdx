## Accessibility

The `Calendar` component follows accessibility best practices:

- Supports keyboard navigation with arrow keys, Tab, Enter, and Space
- Provides proper ARIA labels and descriptions for screen readers
- Maintains focus management throughout date selection
- Uses semantic HTML structure for better accessibility
- Supports screen reader announcements for date changes
- Includes proper role attributes for calendar functionality

## Best Practices

- Use appropriate selection modes based on your use case
- Provide clear visual feedback for selected dates
- Consider disabling invalid dates based on business logic
- Use consistent date formatting throughout your application
- Test with keyboard navigation to ensure accessibility
- Provide clear instructions for date selection when needed
- Consider mobile responsiveness for touch interactions
