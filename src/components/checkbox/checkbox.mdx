## Accessibility

The `Checkbox` component follows the [WAI-ARIA design pattern](https://www.w3.org/WAI/ARIA/apg/patterns/checkbox/) and includes:

- Proper keyboard navigation with Space key to toggle
- Screen reader support with appropriate ARIA attributes
- Focus management and visual focus indicators
- Support for indeterminate state announcements
- Proper labeling associations with `htmlFor` and `id`

## Best Practices

- Always provide a clear, descriptive label for each checkbox
- Use the `required` attribute for mandatory form fields
- Group related checkboxes using fieldsets and legends
- Consider using indeterminate state for parent checkboxes when children have mixed states
- Ensure sufficient color contrast for accessibility
- Test with keyboard navigation and screen readers
