## Accessibility

The `Tabs` component follows the [WAI-ARIA Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabpanel/) and includes:

- Keyboard navigation with arrow keys, Home, and End
- Screen reader support with proper ARIA attributes
- Focus management between tab triggers and content
- Support for both automatic and manual activation modes
- Proper semantic structure with tab roles

## Best Practices

- Use clear, descriptive labels for tab triggers
- Keep tab content organized and scannable
- Consider using manual activation for tabs with expensive content
- Ensure tab content is accessible via keyboard navigation
- Test with screen readers to verify proper announcements
- Use vertical orientation when space is limited horizontally
- Avoid too many tabs in a single group for better usability
