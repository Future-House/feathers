## Accessibility

The `Textarea` component includes:

- Proper labeling support with `htmlFor` and `id` associations
- ARIA attributes for validation states and descriptions
- Keyboard navigation and focus management
- Screen reader compatibility
- Support for required field indicators

## Best Practices

- Always provide clear labels for textarea components
- Use placeholder text to provide helpful input examples
- Set appropriate row heights based on expected content length
- Implement proper validation and error messaging
- Consider character limits for user-generated content
- Use read-only state for content that users should see but not edit
- Test with screen readers to ensure accessibility
- Provide clear visual indicators for required fields
