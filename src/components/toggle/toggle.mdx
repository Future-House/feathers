## Accessibility

The `Toggle` component follows the [WAI-ARIA Button Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/button/) and includes:

- Keyboard support with Space and Enter keys
- Screen reader announcements for pressed state
- Proper ARIA attributes including `aria-pressed`
- Focus management and visual focus indicators
- Support for custom ARIA labels

## Best Practices

- Provide clear visual feedback for pressed/unpressed states
- Use meaningful icons and labels that clearly indicate the toggle's purpose
- Group related toggles using ToggleGroup when appropriate
- Ensure sufficient color contrast for all states
- Test with keyboard navigation and screen readers
- Use appropriate sizes based on the interface context
- Consider the toggle's purpose: use for on/off states, not for navigation
