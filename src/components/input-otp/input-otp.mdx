## Accessibility

The `InputOTP` component follows accessibility best practices and includes:

- Keyboard navigation between input slots with Arrow keys
- Screen reader support with proper ARIA labels
- Focus management within the input group
- Paste support for complete codes
- Clear visual focus indicators
- Proper labeling and instructions
- Support for assistive technologies

## Best Practices

- Provide clear instructions about the expected code format
- Use appropriate input lengths (4-8 digits typically)
- Include visual separators for longer codes
- Implement auto-advance between slots for better UX
- Show validation feedback immediately
- Support paste functionality for convenience
- Use numeric input mode for numeric-only codes
- Provide resend options with appropriate timing
- Test with various input methods and assistive technologies
- Consider auto-submission when code is complete
- Implement proper error handling and retry mechanisms
