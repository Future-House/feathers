## Component API

### ThemeProvider

Theme provider that manages light/dark/system theme state and applies the appropriate CSS classes. It provides a context that can be consumed by child components to access and modify the current theme.

## Accessibility

The `ThemeProvider` component follows accessibility best practices:

- Respects user's system preference when using "system" theme
- Provides smooth transitions between theme changes
- Maintains color contrast ratios according to WCAG guidelines
- Preserves focus states and keyboard navigation across theme changes

## Best Practices

- Wrap your application root with `ThemeProvider` to ensure all components have access to theme context
- Use "system" as the default theme to respect user preferences
- Store theme preference in localStorage to persist across sessions
- Use the `useTheme` hook to access and modify theme state in child components
- Test your application in both light and dark themes to ensure proper contrast and readability
- Consider providing a theme toggle component for users to manually switch themes
